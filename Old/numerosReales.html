Conceptos Técnicos Explicados

¿Qué es la MEMORIA RAM?

Definición: Memoria de acceso aleatorio donde se almacenan temporalmente los datos que la CPU está procesando.

// Cálculo RAM para 10,000 usuarios:
OS: 1.0 GB (Linux optimizado)
PostgreSQL: 4.0 GB (shared_buffers)
App Go: 2.5 GB (10k usuarios activos)
Redis: 0.5 GB (cache)
Buffer: 1.0 GB (margen seguridad)
TOTAL: 9.0 GB REQUERIDOS

Impacto: Más RAM = Más datos en cache = Menos acceso a disco = Mayor velocidad

¿Qué son los HILOS/THREADS?

Definición: Unidades de procesamiento que permiten ejecutar múltiples tareas simultáneamente.

// Cálculo hilos necesarios:
Usuarios concurrentes: 1,000
Requests por usuario: 0.3/segundo
Total RPS: 1,000 × 0.3 = 300
Throughput por core: 1,000 RPS
Cores necesarios: 300 / 1,000 = 0.3 ≈ 1 core

Go vs Otros: Go usa goroutines (hilos ligeros) que consumen ~2KB vs Java/Python que usan ~1MB por hilo

¿Qué son los IOPS?

Definición: Operaciones de Entrada/Salida por Segundo que mide la velocidad de lectura/escritura del disco.

// Cálculo IOPS PostgreSQL:
Transacciones/segundo: 200
WAL writes: 1 IOPS × 200 = 200 IOPS
Data writes: 3 IOPS × 200 = 600 IOPS
TOTAL: 800 IOPS REQUERIDOS

// Comparativa:
HDD: 100-200 IOPS → ❌ INSUFICIENTE
SSD SATA: 50,000 IOPS → ✅ 1.6% uso
SSD NVMe: 500,000 IOPS → ✅ 0.16% uso

¿Qué es la LATENCIA?

Definición: Tiempo que toma una solicitud en viajar desde el cliente hasta el servidor y regresar.

// Desglose latencia total:
Red interna: 2-5ms
Procesamiento App: 5-10ms
Base de datos: 10-20ms
API MikroTik: 10-15ms
TOTAL: 27-50ms

// Límites humanos:
• 0-100ms: Instantáneo
• 100-300ms: Percibible
• 300ms+: Lento
